## 소개
3차원 대장간은 **숭실대학교 창의적공학설계** 강의에서 진행된 **다중 3D 프린터 관리 시스템 구축** 프로젝트입니다. 단순한 3D 프린터 제작을 넘어, 다수의 프린터를 원격으로 통합 조작 및 관리할 수 있는 시스템을 구현하는 것이 목표였습니다. 개발은 하드웨어, 라즈베리파이 소프트웨어, 웹 백엔드, 웹 프론트엔드, AI 서버의 5가지 분야로 나뉘었으며, 저는 이 중 시스템의 핵심 로직을 담당하는 웹 백엔드와 웹 프론트엔드 개발을 전담했습니다.

백엔드 개발에서는 현재까지 사용해 온 NestJS 기반 위에, Spring Boot에서 중요하게 다루는 모범 사례 및 설계 사상을 적극 반영하여 유지보수성이 최대화된 아키텍처를 구축했습니다. 시스템의 안정성을 위해 DTO Converter, ErrorCode, SuccessCode를 표준화하고 Class Validator를 적용하여 모든 API 응답과 요청에 대한 일관성과 안정성을 확보했습니다. 특히, 라즈베리파이 소프트웨어와의 실시간 통신이 핵심인 프로젝트의 특성상 WebSocket을 구현하여 프린터 상태 모니터링, 원격 조작, 파일 업로드 기능을 구현했습니다. 복잡해질 수 있는 서비스 간의 의존성 문제를 해결하기 위해 Pub/Sub 패턴을 성공적으로 도입하여 모듈 간의 결합도를 낮추었으며, 기존의 Notion 기반 문서화를 Swagger로 전환하여 API 명세 파악 및 협업 효율성을 크게 향상시켰습니다.

비록 3D 프린터라는 일반적인 소재로 프로젝트를 진행했지만, 이 경험의 진정한 핵심은 단순히 기능 구현을 넘어 **기술적 도약**에 있었습니다. 저는 지금까지 배운 (NestJS) 지식 위에 **새로운 설계 이론(Spring Boot 스타일의 일관된 구조, Pub/Sub 패턴)**을 과감히 적용하며 기술적 깊이를 더했습니다. 또한, 하드웨어, AI 서버 등 다양한 분야의 팀원들과 끊임없이 소통하고 협업하며, 이 과정에서 일관된 API 응답 형식과 코딩 컨벤션을 유지하는 것의 중요성을 깨달았습니다. 이러한 경험을 통해 새로운 기술 적용 능력과 더불어, 성공적인 팀 프로젝트를 이끌어가는 개발자의 협업 역량을 효과적으로 기를 수 있었습니다.
